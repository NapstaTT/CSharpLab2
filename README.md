# Лабораторная работа №2: Объектно-ориентированное программирование на C#

## Описание проекта

Данный проект представляет собой реализацию лабораторной работы по объектно-ориентированному программированию на языке C#. Работа включает в себя решение задач на создание и использование классов, работу с конструкторами, свойствами, методами и взаимодействием между объектами.

## Вариант: 5

### Решенные задания:

**Задание 1:**
1. `Точка` - класс, представляющий точку на двумерной плоскости
3. `Имя` - класс для представления имени человека с фамилией, именем и отчеством

**Задание 2:**
1. `Линия` - класс, представляющий линию на плоскости с начальной и конечной точками

**Задание 3:**
3. `Город` - класс для представления города и путей к другим городам с стоимостями

**Задание 4:**
8. `Город (модифицированный)` - улучшенная версия класса City с разными конструкторами

**Задание 5:**
5. `Дробь` - класс для работы с дробями и выполнения арифметических операций

## Ключевые особенности реализации

### 1. Модульная архитектура
Проект разделен на логические модули:
- **Services** - служебные классы для взаимодействия с пользователем
- **Classes** - основные сущности предметной области

### 2. Дружественный интерфейс
- Интерактивное меню с выбором заданий
- Проверка корректности ввода данных
- Четкое форматирование вывода

### 3. Реализация требований ООП
- Инкапсуляция (приватные поля, публичные свойства)
- Конструкторы с различными параметрами
- Переопределение метода ToString()
- Взаимодействие между объектами

## Классы и их функциональность

### 1. Point (Точка)
- Представляет точку на плоскости с координатами X и Y
- Конструктор принимает оба параметра (X, Y)
- Метод ToString() возвращает строку в формате "X;Y"

### 2. Name (Имя)
- Представляет имя человека с поддержкой фамилии, имени и отчества
- Три конструктора для различных комбинаций параметров
- Метод ToString() формирует строку в традиционном формате

### 3. Line (Линия)
- Представляет линию с начальной и конечной точками
- Использует объекты класса Point
- Метод ToString() возвращает описание линии

### 4. City (Город)
- Представляет город с возможностью задания путей к другим городам
- Два конструктора: только с названием и с названием + путями
- Метод AddPath() для добавления пути к другому городу с указанием стоимости
- Метод ToString() выводит информацию о городе и его связях

### 5. Fraction (Дробь)
- Представляет математическую дробь с числителем и знаменателем
- Реализует основные арифметические операции (+, -, *, /)
- Поддерживает операции с другими дробями и целыми числами
- Автоматическое упрощение дробей
- Метод ToString() возвращает дробь в формате "числитель/знаменатель"

## Особенности реализации

### 1. Проверка ввода
Класс `InputService` обеспечивает валидацию пользовательского ввода, предотвращая некорректные операции.

### 2. Гибкие конструкторы
Каждый класс имеет несколько конструкторов для создания объектов разными способами, что соответствует принципам перегрузки методов.

### 3. Неизменяемость
Некоторые классы (например, Fraction) реализованы так, что операции возвращают новые объекты, а не изменяют существующие.

### 4. Инкапсуляция данных
Все поля классов являются приватными, доступ к ним осуществляется через публичные свойства.

## Технические детали

### Обработка исключений
- Проверка деления на ноль в классе Fraction
- Валидация входных данных в сервисах

### Форматирование вывода
- Единообразный формат вывода для всех классов
- Четкое разделение результатов разных заданий

### Расширяемость
- Архитектура позволяет легко добавлять новые классы и функциональность
- Четкое разделение ответственности между компонентами
